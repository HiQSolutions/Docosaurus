5. Struttura dell’interfaccia utente
L’interfaccia è suddivisa in quattro sezioni principali:
5.1 Upload & Process
Finalità
caricamento del file Excel CMDB
validazione dei fogli richiesti
esecuzione della trasformazione
Funzionalità principali
rilevamento automatico dei fogli Excel
controllo dei fogli obbligatori
visualizzazione delle dimensioni dei dati sorgente
avvio esplicito del processo da parte dell’utente
avanzamento visivo delle fasi di elaborazione
Fasi di elaborazione
caricamento asset
parsing dei metadati
estrazione inventario applicativo
estrazione inventario infrastrutturale
5.2 View Data
Finalità
consultazione dei dati originali e trasformati
Funzionalità
selezione tra dati originali e processati
visualizzazione per singolo foglio o output
pulizia automatica dei dati per la visualizzazione
gestione colonne duplicate e tipi complessi
metriche su righe, colonne, dimensione e percentuale di null
Opzioni di download
singoli fogli
file Excel combinato con inventari applicativo e infrastrutturale
5.3 Database Upload
Finalità
caricamento degli inventari CMDB nel database
Configurazione (sidebar)
tipo di database
server / host
nome database
credenziali
nomi delle tabelle di destinazione
modalità di caricamento (Append / Replace)
Flusso operativo
test opzionale della connessione
caricamento inventario applicativo
caricamento inventario infrastrutturale
monitoraggio dello stato
riepilogo finale del caricamento
Il caricamento è esplicito e controllato, senza operazioni parziali non tracciate.
5.4 Logs & History
Finalità
tracciabilità operativa e audit
Eventi registrati
avvio sessione
caricamento file
trasformazione dati
caricamento su database
errori ed eccezioni
pulizia manuale dei log
Caratteristiche
log mantenuti in memoria di sessione
massimo 100 eventi
esportazione in formato CSV
6. Gestione errori e sicurezza del dato
blocco del processo se mancano fogli CMDB obbligatori
validazione delle credenziali prima del caricamento
visualizzazione esplicita degli errori
nessun caricamento parziale silenzioso
protezione dello stato di sessione tra le fasi
7. Posizionamento nella Knowledge Base
Questo componente appartiene alla sezione:
Acquisizione Dati → Acquisizione Dati CMDB
Ruolo principale:
punto standard di ingestione delle esportazioni CMDB
normalizzazione dei dati di configurazione
base per analisi applicative e infrastrutturali
integrazione con pipeline ETL e sistemi di reporting
